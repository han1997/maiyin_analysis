# 修复异常旧版 XLS 导入兼容性

## Goal

让应用直接导入公安/旅馆系统生成的旧版 `.xls` 文件，包括工作表声明了大量格式化空行但实际数据很少的 BIFF8 文件，不要求用户另存为 `.xlsx`。

## Requirements

- 保持现有 `.xlsx`、`.xls`、`.csv` 导入行为兼容。
- `内宾综合查询数据导出 - 副本.xls` 必须能够读到 `sheettest` 中的表头和数据行。
- 解析时忽略只有格式、没有单元格内容的尾部空行。
- 文件确有工作表数据时，不得错误提示“没有可读取的数据工作表”。
- 证件号码等业务校验继续由现有导入规则负责，不通过修改用户文件规避解析问题。
- 对该类旧版 XLS 增加自动化回归测试。

## Acceptance Criteria

- [x] 原始副本 `.xls` 无需转换即可进入现有字段识别和记录校验流程。
- [x] 本地集成测试复现修复前的空工作表误判，并在修复后读出 2 行数据。
- [x] 模板 `.xlsx` 的 Calamine 主读取路径保持不变。
- [x] 现有 Rust 测试、格式检查和静态检查通过。
- [x] 解析失败时仍返回清晰、准确的文件级错误。

## Definition of Done

- 测试覆盖异常 XLS 和既有格式。
- `cargo fmt --check`、`cargo check`、相关测试通过。
- 不要求用户修改或重新保存源文件。

## Technical Approach

先用 Calamine 直接复现该 BIFF8 工作簿的范围读取结果，定位是范围元数据、单元格记录还是 API 使用方式导致空范围。优先采用同一依赖内的兼容读取方式；若 Calamine 无法读取该生成器产生的记录，则增加仅针对旧版 XLS 的受控回退解析路径，并复用现有行级字段识别逻辑。

## Decision (ADR-lite)

**Context**：样本是合法 OLE/BIFF8 文件，`sheettest` 实际含 2 行、30 列内容，但文件声明约 46950 行并带大量格式化空行，当前 Calamine 路径返回空数据。

**Decision**：在应用内部兼容源文件，而不是要求用户另存格式；回退范围仅限 `.xls` 读取层。

**Consequences**：旧系统导出文件可直接使用；需要控制额外依赖、内存占用和异常文件处理边界。

## Out of Scope

- 自动修正错误身份证号码。
- 修改源 `.xls` 文件内容。
- 改变入住不足 10 分钟、去重等业务过滤规则。

- 当前依赖：`calamine = 0.36.0`。

- Python `xlrd 2.0.2` 能读取工作表及内容，证明数据记录实际存在。
