# 修复导入记录并美化数据视图切换

## Goal

修复主页“导入记录”在数十万条入住数据下长时间无响应的问题，并将“人员研判 / 导入记录”改造成与现有办公工具风格一致、状态清晰的视图切换控件。

## Confirmed Requirements

- 导入记录必须继续表示“当前分析时间范围内的入住记录”。
- 导入记录由后端分页查询，前端和 Tauri IPC 每次只处理当前页，不再全量读取、反序列化、传输和渲染。
- 默认每页 50 条，提供上一页、下一页、总数和当前页信息。
- 切换会话、合并、重新分析、删除或清空工作区后，导入记录回到第 1 页并重新加载。
- 加载时使用表格骨架或局部加载状态，不用全局阻塞遮罩代替真实分页。
- “人员研判 / 导入记录”使用紧凑的分段式标签，加入一致的线性图标、数量信息、hover、focus 和选中状态。
- 标签使用标准 `tablist` / `tab` / `tabpanel` 语义与 `aria-selected`。
- 视觉保持浅色、克制、可靠的办公工具风格，不使用渐变、重阴影和装饰性动画。
- 旧 SQLite 数据无需迁移，用户已确认可以删除后重新导入原始文件。

## Root Cause

当前 `get_imported_records` 会读取当前会话的全部 `Record`，全量 JSON 反序列化后再按分析时间过滤，并通过 Tauri IPC 返回完整数组。React 随后一次性渲染全部表格行。对于 45 万级记录，该链路同时放大 SQLite 读取、JSON 解码、IPC 序列化和 DOM 渲染成本，表现为模块长期加载不出来。

## Technical Approach

- 为 SQLite `records` 增加可排序、可索引的 `check_in` 列，写入标准时间文本，缺失时间写入 `NULL`。
- 新增 `ImportedRecordsQuery` 和 `ImportedRecordsPage` 跨层 DTO。
- 在 SQLite 中按 `session_id`、分析开始/结束时间完成总数统计和分页，按入住时间、UID 稳定排序。
- Tauri 命令只返回当前页 JSON 明细和总数。
- React 独立维护导入记录页码、页面数据和加载状态；第一次打开和翻页均走分页 API。
- 浏览器演示 API 返回一致的分页结构。
- 数据库 schema 升级采用空库重建策略，不做旧数据回填或兼容迁移。

## UX Direction

使用浅色 `surface-muted` 作为紧凑分段容器。当前标签使用内容面、细边框和深蓝强调；非当前标签保持中性，并提供清晰 hover/focus。人员研判显示人员总数，导入记录显示当前分析范围内的记录总数；数量使用小型徽标并启用等宽数字。加载只发生在导入记录结果区域，用户仍可切回人员研判。

## Acceptance Criteria

- [x] 打开 45 万级会话的导入记录时，单次 IPC 最多返回固定页大小的记录，不返回全量数组。
- [x] 当前分析时间开始和结束边界在 SQLite 查询中正确生效，缺失入住时间的记录不进入结果。
- [x] 总数、上一页、下一页和会话切换后的页码重置正确。
- [x] 导入记录加载和空状态清晰，页面不会因创建数十万 DOM 行而卡死。
- [x] 两个视图标签的视觉语言与现有按钮、表面、颜色 token 一致。
- [x] 标签具备标准 tab 无障碍语义、选中状态和键盘焦点态。
- [x] Rust 单元测试覆盖记录分页、时间范围、排序与页大小限制。
- [x] 前端测试、lint、TypeScript 构建、Rust fmt/test/clippy 通过。

## Out of Scope

- 不修改分析规则、风险评分和导出语义。
- 不增加导入记录搜索或复杂筛选。
- 不迁移或回填旧 SQLite 历史数据。
- 不改动原始 Excel / CSV 文件。
