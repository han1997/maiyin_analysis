# 简化分析参数并优化结果表交互

## Goal

让分析模式、分页、二级弹窗和人员表格更容易理解与操作，修复当前弹窗无法外部关闭、筛选内容被裁切以及列宽错位问题。

## What I already know

- 当前 Rust 通过“是否填写开始/结束时间”隐式决定频次模式，界面没有明确模式字段。
- 只要任一时间边界存在，就启用选定范围频次并停用滚动 7/30/365 天频次。
- 默认设置没有时间边界，因此当前实际默认行为就是高频入住阈值模式。
- 人员页和导入记录页目前都固定为每页 50 条。
- “更多筛选”和“导出”使用原生 `details`，没有外部点击或 Escape 关闭逻辑。
- 更多筛选弹窗位于 `.results-region` 内，而父容器 `overflow: hidden`，导致弹窗内容被裁切。
- 人员表有 10 列，但 CSS 仍按旧的 9 列 `nth-child` 设置宽度，当前第 6 列“365 天”错误获得 220px 宽度。

## Requirements (evolving)

- 分析参数提供两个互斥的模式选择：“选定入住时间范围”和“高频入住阈值”。
- 默认选中“高频入住阈值”，未选中的参数组禁用并降低视觉强调。
- 选择“选定入住时间范围”时，开始时间和结束时间均为必填，且开始时间不得晚于结束时间。
- 模式必须进入前后端 `AnalysisSettings` 契约，Rust 按显式模式分析，不再依赖时间字段是否为空猜测模式。
- 人员研判和导入记录均支持每页 50、100、200 条，默认 50；改变数量后回到第 1 页。
- 更多筛选、导出在点击弹窗外部或按 Escape 时关闭。
- 两个二级弹窗不应同时保持打开；打开其中一个时关闭另一个。
- 更多筛选在常见桌面窗口高度下可完整访问，内容区域允许内部滚动且不被结果容器裁切。
- 人员表使用语义化列 class 设置宽度，不再用易错的全局 `nth-child`；365 天列保持紧凑，预警依据列获得主要弹性空间。
- 视觉继续使用浅色、克制的办公工具风格，所有选中、禁用、hover 和 focus 状态清晰。

## Technical Approach

- 在 TypeScript/Rust `AnalysisSettings` 增加 `frequencyMode: "rolling" | "selected"`，默认 `rolling`，serde 为旧数据提供默认值。
- 使用显式模式控制 `within_analysis_time_window` 和频次预警分支。
- 抽取页面数量选项常量，人员查询和导入记录查询各自维护 page size。
- 将两个 `details` 改为受控 `open` 状态，用文档级 pointerdown/keydown 处理外部点击和 Escape，并保证互斥。
- 调整结果容器与弹窗层级/overflow；筛选弹窗宽高采用视口约束和内部滚动。
- 为人员表各列添加 `people-col-*` class，分别设置固定/弹性宽度，避免影响导入记录表。

## Acceptance Criteria (evolving)

- [x] 新会话和默认设置显示“高频入住阈值”已选中。
- [x] 两个分析模式互斥，未选参数不会参与 Rust 时间过滤或频次评分。
- [x] 选定时间范围模式缺少任一时间、或开始晚于结束时，前端不提交并给出明确提示；Rust 同样拒绝无效直接调用。
- [x] 旧设置缺少模式字段时根据既有时间边界安全推断原分析模式。
- [x] 两张表都可选择 50、100、200 条/页，切换后回到第 1 页并重新请求。
- [x] 点击更多筛选或导出弹窗外部、按 Escape、打开另一个弹窗时，当前弹窗正确关闭。
- [x] 更多筛选全部内容可通过滚动访问，不被结果区域或窗口底部裁切。
- [x] 人员表 365 天列宽紧凑，预警依据、人员、户籍地和操作列宽合理。
- [x] 前端交互测试、Rust 分析测试、lint、build、fmt、clippy 全部通过。

## Definition of Done

- 前后端模式字段与默认值一致。
- 新增模式、分页数量、弹窗关闭和列 class 回归测试。
- 更新相关 Trellis 跨层与前端交互规范。
- 不改变重合入住、同日多次入住的评分公式。

## Out of Scope

- 不新增自定义分页数字输入，页面数量只提供 50、100、200。
- 不重新设计整张人员表或新增列拖拽缩放。
- 不改变导出文件格式和更多筛选业务含义。

## Technical Notes

- 主要文件：`src/App.tsx`、`src/styles.css`、`src/domain/types.ts`、`src-tauri/src/model.rs`、`src-tauri/src/analysis.rs`。
- 视觉场景：公安办公人员在白天桌面环境长时间扫描密集表格，优先清晰、稳定和低干扰。
