# Design

## Style

办公场景为白天室内，用户长时间查看表格和核查证据。界面采用浅色、低刺激、稳定的产品工具风格，不采用指挥大屏视觉。

## Color

使用 OKLCH tokens 管理冷色中性色和单一深蓝强调色：

- 背景：低色度冷灰。
- 内容面：带轻微蓝色相的近白色，不使用纯白。
- 文字：带蓝色相的深灰，不使用纯黑。
- 强调色：低饱和深蓝，仅用于主要操作、选择和焦点。
- 风险色：红、琥珀、蓝、绿分别对应高风险、中风险、关注、正常，同时保留文字标签。

## Typography

使用 `Microsoft YaHei UI`、`Segoe UI` 和系统字体回退。数字启用 tabular figures，表格保持紧凑固定字号，不使用展示字体。

## Components

- 紧凑顶栏：产品名称、本地处理状态、模板和清空动作。
- 左侧控制区：导入、历史、合并、分析口径和数据目录。
- 汇总带：有效入住、人员、预警、高风险和数据问题。
- 结果表：人员维度信息、频次、预警依据、分数和等级。
- 详情检查器：人员信息、预警说明和住宿证据，不以模态框阻断主表。
- 结果筛选：在结果表上方的次级筛选面板中编辑旅馆名称与辖区、户籍包含/排除、年龄、性别和预警状态；这些条件不改变风险评分。
- 参数侧板：只编辑入住时间范围和频次阈值，应用后重新计算风险。
- 状态：骨架加载、空工作区、局部进度、无搜索结果、错误提示和危险确认。

## Layout

桌面端采用可折叠左侧控制区和自适应结果工作区。结果表在窄窗口保留横向滚动，不压缩关键列。人员详情从右侧展开；更窄窗口下占满可用宽度。

## Motion

交互反馈使用 150 至 220 毫秒的 opacity 和 transform 过渡。没有装饰性入场动画，支持 `prefers-reduced-motion`。
